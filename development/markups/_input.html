Но все-таки это еще не готовый к работе виджет, а просто демонстрация выпадайки. Есть детали, связаныне с пересечением классов заполнености и фокуса, есть разные плейсхолдеры и т.д. Чтобы завершить это альтернативный селект делаем пару новых виджетов .input--scanner и .input--picker.


.input--scanner

Редактируемый инпут с выпадайкой. Пока что это просто дефолтный инпут с дефолтной выпадайкой. Особенности проявятся ниже когда дойдем до динамичных лейблов. Алсо это заготовка для датапикера в котором дату, по-идее, можно печатать.


.input--picker

Нередактируемый инпут с выпадайкой. Добавляем атрибут readonly. Курсор превратится в палец. По-сути у нас получается кастомный селект.




.input--adjustable

Unfortunately there is no way to set width of <input> or <select> based on content inside. Nothing like inline-block will work here. Browsers just give <input>'s default widths around 120px (that we usually change to 100%) and that's it. But in some cases we need this width to be changed dynamically based on text inside. So we have to calculate width manually and set it as inline style.

What I suggest here is to create an invisible <div> (see .size-checker in the end of the page). Then copy current text value (or placeholder value if empty) into it. Then copy similar styles to make sure it looks similar. Then just check it .outerWidth() of that element and use it. See sample in the ../scripts/_input.js
