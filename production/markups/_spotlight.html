Details

We add spotlights manually. So each component is responsible for positioning of spotlight itself. In the example above we have .spotlight is mixed to .foobar__help and this element creates the area where spotlight will be stretched. In this example it is 80x80 circle.

Once the area for set light is set the beam with white background will be stretched to it. Tooltip is also being added on top.

Fading curtain is being positioned via position: fixed. It's also inside of .spotlight.

All of the elements described are inside of .spotlight. What we need to do is just to fadeIn() or fadeOut() the whole things. By default, there is display: none;. See .spotlight

But there is an issue. All this spotlight thing lies on top of the entire website, including the element we want to highlight. If nothing is done, then the white spot of spotlight beam will also overlap that element. So we need to do one more thing. Let's take the element we want to highlight and add .under-the-spotlight at the moment of highlighting.
