Intro

Whole mechanism is based on 3 layers:
1) Website layer.
2) .spotlight component that lies overlaps whole website. This component includes fading overlay, light beam and tooltip.
3) .under-the-spotlight class that is being added to the thing we want to highlight lifting up it's z-index.



.spotlight

We add spotlights manually. No libraries are being used. Each component is responsible for positioning of spotlight itself. In the example we have .foobar component and inside of it we have .foobar__help responsible for .spotlight positioning.

Also this component is responsible for how the spotlight beam is going to look. In this example it is 80x80 white circle. See style of .foobar component on this page.

All of the elements including fading curtain, beam and tooltip are packed inside of .spotlight component. To show/hide one or jump to the next one we only need to manipulate .spotlight--current class. Just add and remove it. See ../scripts/_spotlight.js



.under-the-spotlight

But there is an issue. This spotlight component lies on top of the entire website creating the second layer. It overlaps everything on the first level including the thing we want to highlight. So we need to take that thing to the third level adding .under-the-spotlight class.
