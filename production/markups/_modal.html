Scrollbar issue

But there can be a problem with this scrollbar. Because scrollbar is done via overflow property it cuts everything that tries to get out:

This is pretty tough and there is no simple solution. But at least we can disable it in some cases. By default we assume every modal-body may need scrollbar so overflow-y: auto; set in CSS. But if we run JS calculations and see that we are dealing with modal-body that doesn't need a scrollbar yet tehn we can disable this overflow.

To check it please do something like:
$modalBody[0].scrollHeight <= $modalBody.outerHeight()
and if it's true add class .modal-body--no-overflow

See sample in ../scripts/_input.js (search for $modalBody)

Also note that to run these calculations correctly you must run this script before dropdown appearing on the screen because dropdown can stretch partental scrolling area. So pay attention to scripts order and do not run it on input focus as it used to be. Run it on click only.

Otherwise if there is a lot of content it will be fine anyway because this content will stretch modal window and autocomplite dropdown will go over. So we don't need to disable overflow:

CSS conflicts

Make sure to grab only modal related styles from bootstrap. Don't take built in things like normalize or helper classes and especially those code pieces that styles <a> and so on. This is, unfortunately, still happens if you use web ui generator and choose only modal checkbox. So styles have to be edited manually:

You can take this file ../vendors/bootstrap/bootstrap-modal-only.css



JS conflicts

When popup is open try to place .modal-open on <html> tag instead of <body>. This will prevent page jumping to the top on modal open. There is conflict with heigth: 100% that was set on <body> and necessary for layouting.
